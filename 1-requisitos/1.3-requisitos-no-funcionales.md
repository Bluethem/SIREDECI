
# Requerimientos No Funcionales

## 1. Rendimiento

**RNF-01:** El sistema debe soportar al menos 500 usuarios concurrentes sin degradación del rendimiento.  
**RNF-02:** El tiempo de carga de las páginas no debe superar los 3 segundos bajo condiciones normales.  
**RNF-03:** La base de datos debe ser capaz de almacenar un mínimo de 50 000 denuncias anuales.

---

## 2. Disponibilidad y Confiabilidad

**RNF-04:** El sistema debe tener una disponibilidad mínima del 99.5% del tiempo.  
**RNF-05:** Se deben realizar respaldos automáticos diarios de la base de datos.  
**RNF-06:** La recuperación ante desastres no debe exceder las 4 horas.  
**RNF-07:** La pérdida máxima de datos aceptable es de 1 hora.

---

## 3. Usabilidad

**RNF-08:** La interfaz debe ser intuitiva y de fácil uso.  
**RNF-09:** El diseño debe ser responsive y compatible con dispositivos móviles (Android/iOS).  
**RNF-10:** La interfaz debe cumplir con los estándares WCAG 2.1 Nivel AA.

---

## 4. Escalabilidad

**RNF-11:** La arquitectura deberá ser modular, permitiendo agregar nuevas categorías sin rediseño.  
**RNF-12:** La base de datos debe ser escalable para soportar un crecimiento anual del 30%.

---

## 5. Mantenibilidad

**RNF-13:** El código debe estar documentado según los estándares de la organización.  
**RNF-14:** Se deben registrar logs detallados de todas las transacciones críticas.  
**RNF-15:** Las actualizaciones del sistema deben realizarse sin interrupción del servicio (hot deployment).

---

## 6. Compatibilidad

**RNF-16:** El sistema debe ser compatible con los navegadores Chrome, Firefox, Edge y Safari (últimas dos versiones).  
**RNF-17:** Se deberá integrar con sistemas municipales existentes mediante APIs REST.  
**RNF-18:** La exportación de datos debe realizarse en formatos estándar (JSON, CSV, PDF).
