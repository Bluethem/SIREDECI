
# Requerimientos Funcionales

## 1. Sistema de Notificaciones

**RF-01:** El sistema debe permitir el envío de notificaciones internas a los usuarios sobre el estado de sus denuncias.  
**RF-02:** El sistema debe permitir configurar plantillas de notificación según tipo de evento (registro, actualización, resolución).

---

## 2. Módulo de Ciudadanos

### Autenticación y acceso
**RF-03:** El ciudadano debe poder iniciar sesión usando su DNI y fecha de emisión.  
**RF-04:** Se debe permitir registrar denuncias anónimas sin revelar identidad pública.  
**RF-05:** Se debe implementar doble autenticación para validar denuncias registradas.

### Gestión de denuncias
**RF-06:** El ciudadano podrá registrar una denuncia seleccionando una categoría (basura, alumbrado, inseguridad, baches, etc.).  
**RF-07:** Se deberá permitir añadir una descripción detallada del problema.  
**RF-08:** El sistema permitirá adjuntar hasta 5 archivos (fotografías o videos).  
**RF-09:** El usuario podrá seleccionar manualmente la ubicación del hecho denunciado.

### Seguimiento y consultas
**RF-10:** El ciudadano podrá consultar el estado de sus denuncias (Registrado, En revisión, Resuelta, Rechazada).  
**RF-11:** El sistema mostrará el historial completo de cada denuncia registrada.

### Visualización pública
**RF-12:** El sistema presentará estadísticas públicas de denuncias atendidas según categorías.

---

## 3. Módulo de Personal Municipal

### Gestión operativa
**RF-13:** El personal municipal dispondrá de un dashboard con denuncias asignadas y pendientes.  
**RF-14:** El sistema permitirá asignar denuncias a las áreas responsables.  
**RF-15:** El personal podrá cambiar el estado de las denuncias según la resolución dada.  
**RF-16:** Se podrán adjuntar evidencias fotográficas de la resolución.  
**RF-17:** Se permitirá la reasignación de denuncias entre áreas.  
**RF-18:** El sistema identificará y marcará denuncias duplicadas o vinculadas.

### Comunicación
**RF-19:** El personal podrá enviar mensajes o actualizaciones a los denunciantes.  
**RF-20:** El sistema permitirá solicitar información adicional al ciudadano.

---

## 4. Módulo Administrativo

### Administración del sistema
**RF-21:** El administrador podrá gestionar usuarios del sistema (personal municipal y administradores).  
**RF-22:** Se permitirá gestionar roles y permisos (incluyendo el rol de SuperAdmin).

### Reportes y estadísticas
**RF-23:** El sistema generará un dashboard ejecutivo con KPIs (tiempo promedio de atención, denuncias por categoría, tasa de resolución).  
**RF-24:** Se podrán exportar reportes en formatos PDF, Excel y CSV.  
**RF-25:** El sistema ofrecerá análisis de tendencias por zona geográfica.  
**RF-26:** Se mostrará un ranking de desempeño por áreas responsables.

### Configuración
**RF-27:** Se permitirá la gestión y personalización de plantillas de notificaciones.
